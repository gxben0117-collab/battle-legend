## 📋 快速開始

### 步驟 1：打開遊戲
訪問：https://gxben0117-collab.github.io/battle-legend/

### 步驟 2：打開開發者工具
按 **F12** 鍵，切換到 **Console** 標籤

### 步驟 3：載入測試腳本
複製 `自動化測試腳本.js` 的內容，貼到控制台，按 **Enter**

### 步驟 4：運行測試
```javascript
// 快速測試（10 次，約 2 分鐘）

// 完整測試（100 次，約 20 分鐘）⭐ 推薦

// 壓力測試（1000 次，約 3 小時）
```

## 🔧 自定義配置

### 修改測試次數
```javascript
TEST_CONFIG.rounds = 50;  // 測試 50 次
await runAllTests()
```

### 修改測試關卡
```javascript
TEST_CONFIG.stage = '2-5';  // 測試第 2 章第 5 關
await runAllTests()
```

### 修改倍速
```javascript
TEST_CONFIG.speed = 2;  // 使用 2× 倍速
await runAllTests()
```

### 修改間隔時間
```javascript
TEST_CONFIG.delayBetween = 500;  // 每局間隔 0.5 秒
await runAllTests()
```

## 🎓 進階用法

### 測試所有關卡
```javascript
// 測試第 1 章所有關卡
for (let stage = 1; stage <= 10; stage++) {
  TEST_CONFIG.stage = `1-${stage}`;
  TEST_CONFIG.rounds = 10;
  console.log(`\n測試關卡: ${TEST_CONFIG.stage}`);
  await runAllTests();
}
```

### 測試不同倍速
```javascript
// 測試所有倍速
for (let speed = 1; speed <= 3; speed++) {
  TEST_CONFIG.speed = speed;
  TEST_CONFIG.rounds = 10;
  console.log(`\n測試倍速: ${speed}×`);
  await runAllTests();
}
```

### 長期穩定性測試
```javascript
// 連續測試 5 小時
TEST_CONFIG.rounds = 5000;
TEST_CONFIG.delayBetween = 100;
await runAllTests();
```

### 穩定性評分

### 勝率分析
- **80%+ ✅**：難度合理
- **50-80% ⚠️**：可能偏難
- **50% 以下 ❌**：太難了，需要平衡

### 崩潰分析
- **0 次 ✅**：完美穩定
- **1-5 次 ⚠️**：偶發問題，需要修復
- **5 次以上 ❌**：嚴重問題，優先修復

## 🐛 常見問題

### Q: 測試卡住不動了
A: 檢查是否真的卡住（等待 1 分鐘），或按 F5 刷新重新開始

### Q: 測試結果不準確
A: 確保遊戲頁面在前台，不要切換到其他標籤

### Q: 想提前停止測試
A: 刷新頁面（F5）或關閉標籤

### Q: 測試完成後想重置
A: 刷新頁面，重新貼入腳本

## 💡 使用建議

### 何時測試
- ✅ 每次重大更新後
- ✅ 修復 Bug 後驗證
- ✅ 發布前最終測試
- ✅ 用戶報告問題後

### 測試策略
1. **快速測試**（10 次）- 驗證腳本正常
2. **完整測試**（100 次）- 找出常見問題
3. **修復 Bug**
4. **再次完整測試** - 驗證修復
5. **壓力測試**（1000 次）- 確保長期穩定

### 結果處理
1. 保存測試結果（複製控制台輸出）
2. 分析崩潰和錯誤詳情
3. 修復發現的問題
4. 再次測試驗證

## 📝 報告 Bug

### 提供信息
當發現問題時，請提供：
1. **測試次數**：第幾次測試出現
2. **錯誤信息**：控制台的錯誤詳情
3. **崩潰堆疊**：error.stack 內容
4. **測試配置**：關卡、倍速、模式
5. **復現步驟**：如何重現問題

### 示例報告
